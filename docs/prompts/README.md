# Planning Prompts

This directory contains markdown prompts for kicking off planning mode for the Dual-Mode SDLC Adaptation Agent case-study product.

Use them in this order:

1. `planning-mode-kickoff.md` — starts planning mode, defines product thesis, MVP scope, non-goals, safety boundaries, contracts, and required planning output.
2. `original-manifest-review.md` — reviews the original 20-agent manifest once supplied and classifies each agent as keep, merge, split, delay, or remove from MVP.
3. `agent-execution-waves.md` — recombines the original manifest into safer implementation waves with ownership, outputs, checks, and integration checkpoints.
4. `wave-2-local-agent-mvp.md` — executes Wave 2 by implementing the CLI-first Local Agent MVP commands, fixtures, reports, and checks.
5. `wave-4-execution.md` — executes Wave 4 by packaging reporting, demo assembly, technical integration, and the final case-study narrative.

The prompts intentionally keep MVP work fixture-backed and CLI-first. The planning prompts produce an implementation-ready plan without writing code, while the wave execution prompts turn that plan into scoped implementation work.
